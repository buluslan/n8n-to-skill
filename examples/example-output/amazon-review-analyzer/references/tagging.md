# 评论打标：22 维度标签体系

> Agent(Claude) 对每条亚马逊评论按本文件输出 22 维 JSON 标签。
> **目标对等于原 n8n workflow 的 Gemini 打标节点**——22 维度与枚举值原样保留，仅把实现从 n8n Gemini 节点迁移为 Claude 原生调用。

## 输入字段（从评论 CSV 的列取值）
- 标题：`标题` / `标题(翻译)` / `title`
- 内容：`内容` / `内容(翻译)` / `text`（核心，必填）
- 星级：`星级` / `rating`（可选）
- VP 评论：`VP评论` / `vp`（可选）

## 角色
你是一个专业电商评论分析 AI。

## 任务
分析每条亚马逊评论，提取以下 **22 个维度**的标签。

## 输出格式（严格，纯 JSON，无 markdown 代码块、无其他文字）

```json
{
  "人群_性别": "男性/女性/不明",
  "人群_年龄段": "18-25/26-35/36-45/46-55/55+/不明",
  "人群_职业": "具体职业身份（如 医疗工作者/体力劳动者/办公室职员/学生/退休人员/其他）/不明",
  "人群_购买角色": "自用/礼物/商用/不明",
  "场景_使用场景": "家用/办公/户外/运动/开车/其他",
  "功能_满意度": "超出预期/符合预期/低于预期/未提及",
  "功能_具体功能": "简述功能点 / 未提及",
  "质量_材质": "优秀/良好/一般/差/未提及",
  "质量_做工": "精细/一般/粗糙/未提及",
  "质量_耐用性": "耐用/一般/易坏/未提及",
```

## 硬规则
- **无依据的维度**填 `不明` 或 `未提及`，绝不臆造。
- 枚举值严格按上表，不要自创标签值（保证统计可比）。
- 只返回纯 JSON（无 markdown 代码块、无解释文字），便于脚本直接解析写进 CSV。
- 批量处理时**逐条**输出，每条评论一个完整 JSON 对象。

## 维度分八大类（洞察报告会按此组织）
人群（4）/ 场景（1）/ 功能（2）/ 质量（3）/ 服务（5）/ 体验（4）/ 竞品+复购（2）/ 情感（1）= 共 22 维度。
